# 服务端密码校验改造 — 2026-07-24

## 目标
写/改/删操作需密码保护，前端输入一次后自动带上 header，服务端统一校验。

## 改造内容

### 1. 新建 auth.ts
- `checkPassword(req)` 校验 `x-poem-password` header，失败返回 401
- `PASSWORD_KEY` 定义 localStorage key 名
- 读写 localStorage 的工具函数

### 2. 前端 api.ts 改造
- `apiFetch` 新增 `requireAuth` 参数
- 写操作调用时自动从 localStorage 读密码并带 `x-poem-password` header
- 401 错误抛特定错误对象（`.status=401`），前端可据此弹窗重输

已加 `requireAuth: true` 的函数：
- addPoem
- restorePoem
- updatePoem
- deletePoem
- permanentlyDeletePoem
- deleteCollectionApi
- addCollectionApi
- runBatchAiTags

### 3. 服务端 API 路由加固
以下路由的写操作方法已加入 `checkPassword(req)`：

| 路由 | 方法 | 校验 |
|------|------|------|
| `/api/poems` | POST | ✅ |
| `/api/poem/[id]` | PUT, DELETE | ✅ |
| `/api/poems/[id]` | PATCH | ✅ |
| `/api/collection/[id]` | DELETE | ✅ |
| `/api/poems/batch-tags` | POST | ✅ |

读操作（GET）未加校验，保持公开访问。

## 密码来源
服务端从环境变量读取：
- `POEM_PASSWORD`（优先）
- `NEXT_PUBLIC_POEM_PASSWORD`（fallback）

当前生产环境密码仍为 `zsklj`（硬编码在 `PasswordGate.tsx` 的 fallback 中，已打进前端 bundle）。

## 安全说明
`NEXT_PUBLIC_POEM_PASSWORD` 会打包进前端 bundle，仅为 fallback，不影响服务端校验强度。
前端 localStorage 存储的密码仅用于自动填充 header，无法绕过服务端校验。
