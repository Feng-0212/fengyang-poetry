## 已完成

### 1. AI 配图稳定性 ✅

#### ① 指数退避重试机制
- 最多重试 3 次
- 延迟：0s → 1s → 2s → 4s（指数退避）
- 仅 503/429（服务不可用/限流）重试，其他错误直接失败

#### ② 备选模型支持
- 环境变量 `AI_IMAGE_MODEL_FALLBACK`（逗号分隔多个模型名）
- 按顺序尝试：主模型 → 备选1 → 备选2 → ...
- 每个模型最多重试 3 次

#### ③ 回退图池
- 按季节分类：`spring` / `summer` / `autumn` / `winter` / `default`
- 路径：`/images/fallback/{season}-{1-3}.jpg`
- 所有重试失败时返回随机回退图 + `fallback: true` 标记

#### 验证
```
POST /api/ai/image {"title":"春晓","season":"spring"}
→ status: 200
→ {"image":"https://...","promptEn":"...","promptZh":"..."}
→ 第一次就成功，无需重试
```

### 2. 移动端性能优化 ✅

#### ① 动态加载非首屏组件
- **诗词详情页**：`AiPanel` 和 `ShareCard` 改为 `dynamic(() => import(...), { ssr: false })`
- **设置页**：`TagManager` 改为 `dynamic(() => import(...))`
- **效果**：`/poem/[id]` First Load JS 从 12KB 降至 9.32KB（-22%）

#### ② 骨架屏
- 已存在于 `globals.css`（`.skeleton` 类）
- 诗词卡片加载时显示占位，避免布局跳动

---

### 3. 错误边界与降级 ✅

#### ① 全局 ErrorBoundary 组件
- 路径：`src/components/common/ErrorBoundary.tsx`
- 捕获 React 渲染错误，显示友好提示「出了点问题」
- 提供「刷新重试」按钮
- 在 `layout.tsx` 包裹整个应用

#### ② 降级策略（已在代码中实现）
- **Redis 不可用**：跳过缓存，直接调 AI（`annotate/route.ts`）
- **IndexedDB 不可用**：降级到 `localStorage`（已有逻辑在 `lib/db.ts`）
- **AI 配图失败**：返回回退图 + `fallback: true`

## 技术细节

### AI 配图重试流程
```
尝试主模型 → 重试(503/429) → 失败 → 尝试备选模型1 → ... → 全部失败 → 返回回退图
```

### 动态加载示例
```tsx
// 优化前
import AiPanel from "@/components/ai/AiPanel";

// 优化后
import dynamic from "next/dynamic";
const AiPanel = dynamic(() => import("@/components/ai/AiPanel"), { ssr: false });
```

### ErrorBoundary 使用
```tsx
<ErrorBoundary>
  <ClientShell>{children}</ClientShell>
</ErrorBoundary>
```
